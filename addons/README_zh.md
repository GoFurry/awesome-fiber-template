# 可选增强服务

`addons` 是一个独立的可选能力池，用来放置可复用的服务适配器和工具包占位文档。

它被刻意放在 `v3/` 之外，这样四套模板就可以继续保持各自清晰的工程边界。

## 设计目标

- 每个 addon 都保持尽量小，便于复制或按需引用。
- 所有 addon 都必须是可选项，不和任何模板版本强耦合。
- 尽量做到一个能力一个包。
- 先写清接口和边界，再补实现。

## 建议结构

```text
addons/
  mongodb/
  redis/
  minio/
  kafka/
  rabbitmq/
  mail/
  sms/
  jwt/
  id/
  crypto/
  retry/
  paginator/
  observability/
```

## 接入原则

模板默认不要依赖 `addons`。
只有当项目确实需要某个能力时，再在应用边界进行接入。

## 当前状态

所有 addon 目录目前都只是占位。
每个目录都放了一份文档，用来说明未来的作用和建议形态。
